# PROJECT_STATE

## 项目目标
- 本地可运行的博维 AI 升级项目驾驶舱。
- 跑通闭环：进度/会议输入 -> AI 提取建议 -> 确认中心 -> 写入任务/成果/问题 -> 驾驶舱更新。

## 技术栈
- 后端：FastAPI
- 数据库：SQLite + SQLAlchemy
- 前端：原生 HTML / CSS / JS
- 数据来源：SQLite 种子数据 + Excel 导入

## 关键目录
- `app/main.py`：FastAPI 入口
- `app/models.py`：任务/成果/问题/确认等数据模型
- `app/routers/`：各模块 API
- `app/services/extractor.py`：AI 提取模拟逻辑
- `app/static/app.js`：核心前端页面渲染与交互
- `app/static/style.css`：全局样式
- `app/templates/index.html`：单页入口
- `app/excel_importer.py`：Excel 导入逻辑
- `bowei_ai_dashboard.db`：当前本地数据库

## 重要决策
- 前端坚持原生单页，不引入前端框架。
- `app/static/app.js` 承担主要页面渲染，当前是“数据 + 模板字符串 + 事件函数”模式。
- 成果文件不存本地文件，只存文件链接地址。
- AI 提取先走规则/模拟逻辑，接口保留，后续可替换真实 LLM。
- 问题与决策页、更新页部分内容在接口不足时使用 mock/兜底数据保证展示完整。

## 当前问题
- `app.js` 已经很大，页面逻辑集中，后续维护成本偏高。
- 更新页右侧字段编辑目前主要是前端展示层，未把内联编辑结果完整回写到后端结构化字段。
- 问题与决策页目前偏展示，真正的“改状态 / 写结论 / 关闭问题”闭环还没做完。
- 成果库“待上传”只是提示，未实现正式的链接编辑入口。
- 本地启动有时不稳定，当前可用的兜底方式是 `run_server_tmp.bat`。
