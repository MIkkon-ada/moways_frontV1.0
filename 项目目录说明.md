# 项目目录说明

这份说明只描述当前工作区里真实存在、且仍然有用的内容。

## 当前结构

```text
moways_ai/
├─ bowei_ai_dashboard/   # 后端主项目，FastAPI + SQLite / PostgreSQL
├─ frontend/             # 新的 React + Vite 前端
├─ run-logs/             # 运行日志
├─ *.md / *.docx / *.xlsx # 需求、说明、记录、表格
├─ *.db                  # 本地数据库文件
├─ .env.example          # 环境变量示例
├─ docker-compose.yml    # 本地 PostgreSQL 启动文件
├─ start-dev.bat         # 前后端一键启动脚本
├─ _tmp_*.txt            # 临时文本文件
└─ .gitignore
```

## 关键结论

1. 当前工作区**没有 `app/` 作为主前端目录**。
2. 旧的 HTML/CSS/JS 界面已经从 `bowei_ai_dashboard/app/templates` 和 `bowei_ai_dashboard/app/static` 中移除。
3. 现在唯一正式的前端入口是 `frontend/`。
4. 如果你还在本机看到 `C:\Users\25861\bowei-page1-dashboard.html`，那是工作区外的独立文件，不属于当前仓库。

## `bowei_ai_dashboard/`

后端主项目，只保留 API、认证、数据库和业务逻辑。

典型内容：
- `app/main.py`：后端入口
- `app/database.py`：数据库连接
- `app/models.py`：数据模型
- `app/routers/`：API 路由
- `app/services/`：业务逻辑
- `README.md`：后端运行说明

## `frontend/`

新的正式前端，只负责界面展示和交互。

典型内容：
- `src/main.tsx`：前端入口
- `src/App.tsx`：页面切换
- `src/pages/`：页面级组件
- `src/components/`：通用组件
- `src/data/`：mock 数据
- `vite.config.ts`：开发代理配置

当前页面：
- 首页驾驶舱
- 语音更新
- 会议纪要
- AI 确认中心
- 工作推进表

## 建议的阅读顺序

1. `bowei_ai_dashboard/README.md`
2. `frontend/package.json`
3. `frontend/src/App.tsx`
4. `frontend/src/components/Sidebar.tsx`
5. `frontend/src/data/dashboardMock.ts`

## 启动方式

### 后端

```bash
cd bowei_ai_dashboard
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 前端

```bash
cd frontend
npm install
npm run dev
```

### 访问地址

- 后端：`http://127.0.0.1:8000`
- 前端：`http://127.0.0.1:5173`

## 备注

- `localhost` 和 `127.0.0.1` 不要混用，尤其是涉及登录态时。
- 旧页面如果以后还要留档，建议统一放到 `archive/` 或单独的历史目录里。
